   - 移除硬编码的 `SECRET` 常量

2. ✅ **配置更新**:
   - `service-user/application.yml`: 添加jwt配置节
   - `service-order/application.yml`: 添加jwt配置节  
   - `service-gateway/application.yml`: 添加jwt配置节
   - 所有配置支持环境变量 `${JWT_SECRET:gateway-secret}`

3. ✅ **文档创建**:
   - 创建 `.env.example` 环境变量示例文件
   - 创建 `docs/deployment/环境变量配置指南.md` 部署文档
   - 创建 `docs/decisions/001-JWT-Secret外部化决策记录.md` 决策文档

4. ✅ **验证工作**:
   - 检查所有硬编码的 `gateway-secret` 都已替换
   - 验证配置语法正确
   - 确保向后兼容性（保持默认值）

#### 技术实现
1. **JwtUtil修改**:
   ```java
   // 修改前
   private static final String SECRET = "gateway-secret";
   private static final long EXPIRE = 24 * 3600 * 1000;
   
   // 修改后
   @Value("${jwt.secret:gateway-secret}")
   private String secret;
   @Value("${jwt.expire:86400000}")
   private long expire;
   ```

2. **配置示例**:
   ```yaml
   jwt:
     secret: ${JWT_SECRET:gateway-secret}
     expire: 86400000
   ```

3. **环境变量**:
   ```bash
   # .env.example
   JWT_SECRET=your-strong-jwt-secret-key-here
   ```

## 🔍 问题发现过程

### 1. 用户反馈
用户报告：`com/example/gateway/AuthGlobalFilter.java:23` 不能正常引入 `JwtUtil`

### 2. 问题分析步骤
1. **检查错误代码位置**：`AuthGlobalFilter.java` 第23行 `@Autowired private JwtUtil jwtUtil;`
2. **检查JwtUtil位置**：位于 `common` 模块 (`common/src/main/java/com/example/common/util/JwtUtil.java`)
3. **检查依赖关系**：发现 `service-gateway/pom.xml` 缺少对 `common` 模块的依赖
4. **对比其他模块**：`service-user` 和 `service-order` 都有 common 依赖，只有 gateway 没有

### 3. 根本原因
- `service-gateway` 模块需要引用 `common` 模块中的 `JwtUtil` 类
- 但 `service-gateway/pom.xml` 中没有声明对 `common` 模块的依赖
- 导致编译时找不到 `JwtUtil` 类

## 🛠️ 修复实施

### 1. 修改文件
**文件**: `service-gateway/pom.xml`

**修改内容**：
```xml
<!-- 在 dependencies 部分添加 -->
<dependency>
    <groupId>com.example</groupId>
    <artifactId>common</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>
```

**位置**: 放在 `spring-cloud-starter-loadbalancer` 依赖之后，`</dependencies>` 之前

### 2. 添加注释
为依赖添加说明注释：
```xml
<!-- Common 模块依赖（修复 JwtUtil 引入问题） -->
```

## 🎓 经验教训

### 1. 模块化开发要点
- **依赖声明必须完整**：使用其他模块的代码必须显式声明依赖
- **统一依赖管理**：所有服务模块应该有一致的依赖结构
- **及时验证**：添加新模块或代码后要及时编译验证

### 2. 问题排查流程
1. **定位错误**：精确到文件行号
2. **分析依赖**：检查类所在模块和使用模块
3. **对比验证**：与其他正常模块对比
4. **实施修复**：最小化修改，添加注释
5. **文档记录**：详细记录问题和解决方案

### 3. 文档重要性
- **问题文档**：记录问题现象和分析过程
- **修复文档**：记录修复方案和实施步骤

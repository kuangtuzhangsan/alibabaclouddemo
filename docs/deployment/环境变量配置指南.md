# 🔧 环境变量配置指南

## 📋 概述

本项目使用环境变量进行配置管理，以提高安全性和灵活性。所有敏感信息和环境相关配置都应通过环境变量设置。

## 🚀 快速开始

### 1. 创建环境变量文件
```bash
# 复制示例文件
cp .env.example .env

# 编辑配置文件
vim .env
```

### 2. 设置关键环境变量
```bash
# 生成安全的 JWT Secret
export JWT_SECRET=$(openssl rand -base64 32)

# 设置数据库密码
export MYSQL_PASSWORD=your-strong-password-here
```

### 3. 启动服务
```bash
# 使用环境变量启动
source .env && mvn spring-boot:run -pl service-user

# 或者使用 dotenv
npm install -g dotenv-cli
dotenv -- mvn spring-boot:run -pl service-user
```

## 🐳 Docker 部署

### Docker Compose 示例
```yaml
version: '3.8'
services:
  service-user:
    image: your-registry/service-user:latest
    environment:
      - JWT_SECRET=${JWT_SECRET}
      - MYSQL_HOST=${MYSQL_HOST}
      - MYSQL_PASSWORD=${MYSQL_PASSWORD}
      - NACOS_SERVER=${NACOS_SERVER}
      - REDIS_HOST=${REDIS_HOST}
      - REDIS_PORT=${REDIS_PORT}
    env_file:
      - .env
    ports:
      - "28081:28081"
    depends_on:
      - mysql
      - redis
      - nacos

  service-order:
    image: your-registry/service-order:latest
    environment:
      - JWT_SECRET=${JWT_SECRET}
      # ... 其他环境变量
    env_file:
      - .env
    ports:
      - "8082:8082"

  service-gateway:
    image: your-registry/service-gateway:latest
    environment:
      - JWT_SECRET=${JWT_SECRET}
      # ... 其他环境变量
    env_file:
      - .env
    ports:
      - "28080:28080"
```

### Docker 运行命令
```bash
# 构建镜像
docker build -t service-user -f service-user/Dockerfile .

# 运行容器
docker run -d \
  --name service-user \
  -e JWT_SECRET=your-secret \
  -e MYSQL_PASSWORD=your-password \
  -p 28081:28081 \
  service-user
```

## ☁️ 云平台部署

### Kubernetes 配置示例
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  MYSQL_HOST: "192.168.31.17"
  REDIS_HOST: "192.168.31.17"
  NACOS_SERVER: "192.168.31.17:18848"
  # ... 其他非敏感配置
---
apiVersion: v1
kind: Secret
metadata:
  name: app-secrets
type: Opaque
data:
  JWT_SECRET: base64-encoded-secret
  MYSQL_PASSWORD: base64-encoded-password
  # ... 其他敏感配置
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: service-user
spec:
  template:
    spec:
      containers:
      - name: service-user
        image: your-registry/service-user:latest
        env:
        - name: JWT_SECRET
          valueFrom:
            secretKeyRef:
              name: app-secrets
              key: JWT_SECRET
        - name: MYSQL_PASSWORD
          valueFrom:
            secretKeyRef:
              name: app-secrets
              key: MYSQL_PASSWORD
        - name: MYSQL_HOST
          valueFrom:
            configMapKeyRef:
              name: app-config
              key: MYSQL_HOST
        # ... 其他环境变量
```

## 🔐 安全最佳实践

### 1. JWT Secret 管理
```bash
# 生成强密码
openssl rand -base64 32
# 输出: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

# 在环境中设置
export JWT_SECRET=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

### 2. 密码复杂度要求
- 至少12个字符
- 包含大小写字母
- 包含数字
- 包含特殊字符
- 避免使用字典单词

### 3. 环境分离
```bash
# 开发环境
.env.development

# 测试环境  
.env.test

# 生产环境
.env.production
```
